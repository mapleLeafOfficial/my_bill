# 合租记账本 — 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
三人合租房子，每月产生各种共同费用（水电费、网费、燃气费等），需要记录并平分这些费用。目前可能通过微信群或备忘录记录，不够清晰且容易遗漏。

### 1.2 项目目标
开发一个简洁的 Web 记账应用，实现：
- 记录每笔共同支出
- 自动计算每人应付金额
- 按月结算，清晰展示账目明细
- 提供概览视图，一目了然当前账目状态

### 1.3 目标用户

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 合租室友 | 三位共同租房的人 | 记录支出、查看账目、月度结算 |
| 记账人 | 负责记录费用的人 | 快速录入、修改记录 |

## 2. 功能模块

> 按优先级从高到低排列。

### 模块 M1：支出记录管理（核心）
- **优先级**：P0
- **涉及层级**：前端 + Supabase 数据库
- **描述**：记录、查看、编辑、删除日常支出
- **用户故事**：
  - 作为室友，我希望记录一笔支出（金额、类别、日期、付款人、备注），以便后续结算
  - 作为室友，我希望查看所有支出记录，以便核对账目
- **功能点**：
  - [ ] 新增支出记录（金额、类别、日期、付款人、备注）默认有一些 ， 核心必填的就只有那个 ，
  - [ ] 支出列表展示（按日期排序）
  - [ ] 编辑支出记录
  - [ ] 删除支出记录
  - [ ] 支出类别：水电费、网费、燃气费、物业费、其他

### 模块 M2：概览仪表盘
- **优先级**：P0
- **涉及层级**：前端 + Supabase 数据库
- **描述**：首页展示当前账目概览
- **用户故事**：
  - 作为室友，我希望在首页看到当前月份的总支出和每人应付金额
- **功能点**：
  - [ ] 当前月份总支出
  - [ ] 每人已付金额 / 应付金额
  - [ ] 结算状态（谁欠谁多少钱）
  - [ ] 最近 5 笔支出

### 模块 M3：月度结算
- **优先级**：P0
- **涉及层级**：前端 + Supabase 数据库
- **描述**：按月查看和结算账目
- **用户故事**：
  - 作为室友，我希望按月查看账目明细，并在月底进行结算
- **功能点**：
  - [ ] 月份选择器
  - [ ] 按月展示所有支出
  - [ ] 月度汇总（总支出、每人应付）
  - [ ] 结算建议（谁需要给谁转账多少钱）
  - [ ] 标记月度已结算

### 模块 M4：人员管理 （管理面板 ， 无需认证的那种 可以自定义👤那种 ， 上传图片）
- **优先级**：P1
- **涉及层级**：前端 + Supabase 数据库
- **描述**：管理合租人员信息（固定三人，可修改名字）
- **用户故事**：
  - 作为用户，我希望可以设置三位室友的姓名，以便记录谁付了钱
- **功能点**：
  - [ ] 查看当前人员列表
  - [ ] 修改人员姓名

### 模块 M5：数据统计
- **优先级**：P2
- **涉及层级**：前端 + Supabase 数据库
- **描述**：支出统计和趋势分析
- **功能点**：
  - [ ] 按类别统计支出占比
  - [ ] 月度支出趋势图

## 3. 数据模型设计

### 3.1 核心实体

#### 人员 (members)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | uuid | 主键 | 自动生成 |
| name | varchar(50) | 姓名 | 非空 |
| created_at | timestamp | 创建时间 | 自动生成 |

#### 支出记录 (expenses)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | uuid | 主键 | 自动生成 |
| amount | decimal(10,2) | 金额 | 非空 |
| category | varchar(20) | 类别 | 非空 (水电/网费/燃气/物业/其他) |
| description | varchar(200) | 备注 | 可空 |
| payer_id | uuid | 付款人ID | 外键 -> members.id |
| expense_date | date | 支出日期 | 非空 |
| month | varchar(7) | 所属月份 | 格式 YYYY-MM，用于结算分组 |
| created_at | timestamp | 创建时间 | 自动生成 |
| updated_at | timestamp | 更新时间 | 自动更新 |

#### 月度结算 (settlements)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | uuid | 主键 | 自动生成 |
| month | varchar(7) | 月份 | 格式 YYYY-MM |
| total_amount | decimal(10,2) | 总金额 | 非空 |
| is_settled | boolean | 是否已结算 | 默认 false |
| settled_at | timestamp | 结算时间 | 可空 |
| created_at | timestamp | 创建时间 | 自动生成 |

### 3.2 实体关系

```
members (3条记录，固定)
    ↓ 1:N
expenses (多条支出记录，每条有一个付款人)

settlements (每月一条结算记录)
```

## 4. 技术方案

### 4.1 技术栈
- **前端**：React 18 + Vite + TypeScript
- **UI 框架**：Tailwind CSS + shadcn/ui
- **状态管理**：React Query (TanStack Query)
- **数据库/BaaS**：Supabase (PostgreSQL)
- **图表**：Recharts

### 4.2 Supabase 功能使用
- **数据库**：PostgreSQL 表存储数据
- **实时订阅**：多人同时使用时数据同步
- **Row Level Security**：简化版可暂不启用（内部使用）

### 4.3 API 设计（Supabase 直接操作）

通过 Supabase JS SDK 直接操作数据库：

| 操作 | 表 | 说明 |
|------|------|------|
| 获取所有成员 | members | 初始化时加载 |
| 获取支出列表 | expenses | 支持按月份筛选 |
| 新增支出 | expenses | INSERT |
| 更新支出 | expenses | UPDATE by id |
| 删除支出 | expenses | DELETE by id |
| 获取/更新结算状态 | settlements | 按月份操作 |

## 5. 非功能性需求

- **安全性**：内部使用，无登录需求。可通过 Supabase 匿名访问或简单的访问密码
- **性能**：数据量小（每月约 20-50 条记录），无需特别优化
- **兼容性**：支持主流浏览器，移动端友好（响应式设计）
- **可用性**：部署在 Vercel/Netlify，Supabase 云数据库保证 99.9% 可用
